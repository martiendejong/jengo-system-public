# Centralize vs. Decentralize Decision Tree
# Quick Reference Guide

**Purpose:** Fast decision-making for operation coordination
**Based on:** Coordination Multiplier Framework (Hayek 1945 + Brensen & Hitzig 2025)

---

## 30-Second Decision Protocol

```
NEW OPERATION → Q1: Human authority? YES → DECENTRALIZE
                                      NO  → Q2: Tacit knowledge >3.0? YES → DECENTRALIZE
                                                                       NO  → Q3: Latency >1s? YES → DECENTRALIZE
                                                                                              NO  → Q4: Synergies >5x? YES → CENTRALIZE
                                                                                                                       NO  → Q5: Antifragile? YES → DECENTRALIZE
                                                                                                                                              NO  → HYBRID
```

## Quick Scoring

**Q1: Human Decision Authority?**
- YES → DECENTRALIZE (DONE status, PR merge, destructive ops)
- NO → Continue

**Q2: Tacit Knowledge >3.0?**
- Score = (intuition × context × experience) / explicit_rules
- YES → DECENTRALIZE (gut feelings, Ring 2 uncertainty)
- NO → Continue

**Q3: Latency >1 Second?**
- YES → DECENTRALIZE (real-time, physical distance)
- NO → Continue

**Q4: Coordination Synergies >5x?**
- Synergies = shared_data × global_patterns × batch_efficiency
- YES → CENTRALIZE (Session Restoration 5,035x, Priority Ranking 95.6%)
- NO → Continue

**Q5: Antifragility Critical?**
- YES → DECENTRALIZE (3-Ring SCP distributed veto)
- NO → HYBRID (central coordination + local execution)
